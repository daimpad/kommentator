Kommentator

Willkommen

Markiere im Text eine Stelle, um sie zu kommentieren. Über den Button ☰ unten rechts erreichst du alle Funktionen — Kommentare laden, herunterladen, Hilfe und den ☾/☀ Hell-/Dunkelmodus. Die Notizspalte rechts lässt sich am Griff breiter ziehen.

Das Namensfeld ordnet Kommentare deinem Namen zu — es ist kein Zugriffsschutz. Kommentare bleiben im Browser, bis du sie exportierst.

Beispieltext zum Ersetzen

Dies ist Platzhaltertext. Ersetze den Inhalt dieses Bereichs (<div data-kommentierbar>) durch das Dokument, das kommentiert werden soll. Die Markierungs- und Kommentarfunktion arbeitet auf jedem Fließtext, auch über mehrere Absätze und verschachtelte Elemente hinweg.

Jede Markierung speichert den exakten Wortlaut, etwas Kontext davor und danach sowie die Zeichenposition im Text. So lassen sich Kommentare beim Wiedereinlesen genau an derselben Stelle verankern — vorausgesetzt, der Ausgangstext bleibt zwischen den Runden unverändert.

Für geteilte Sichtbarkeit ohne Server sammelst du die exportierten Dateien der Beteiligten ein und liest sie über „Kommentare laden“ gemeinsam ein. Alle Notizen erscheinen dann nebeneinander in der Randspalte, mit Namen der jeweiligen Autor:innen.

Einbindung & Sicherheit

So bindest du es ein

Drei Schritte, kein Build, keine Abhängigkeiten:

<link rel="stylesheet" href="kommentare.css">
<div data-kommentierbar> … dein Fließtext … </div>
<script src="kommentare.js"></script>
<script>
  Kommentare.init({
    container: '[data-kommentierbar]',
    autor: 'Vorname Nachname',
    toolbarMode: 'floating', // Button unten rechts statt Balken oben
    themeToggle: true
    // weiter: help, resizable, texte, readOnly,
    //         onCreate/onUpdate/onDelete/onChange/onThemeChange
  });
</script>

Sicherheit & geteilte Sichtbarkeit

Das Namensfeld dient nur der Zuordnung, es ist kein Zugriffsschutz. Echten Schutz regelt der Betrieb serverseitig, z. B. per HTTP Basic Auth (.htaccess). Geteilte Sichtbarkeit ist asynchron und ohne Backend: exportieren → einsammeln → über „Kommentare laden“ importieren. Beim Export werden source (URL) und sourceTitle gespeichert.
